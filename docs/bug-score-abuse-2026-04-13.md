# 버그 #3 — 점수 이상 (36점 사례)

**발견일**: 2026-04-13
**세션 실측**: `agent_interview_sessions.id = f04d1416`, message_index=1

## 증상

무의미 반복 답변("제일제일제일 어려웠던 ...")에 evaluator가 **overallScore=36**을 매김. 사용자는 "개떡인데 36점은 이상하다"고 판단.

실측 scores:
```
{ clarity: 30, accuracy: 50, practicality: 40, depth: 20, completeness: 30 }
→ 30*0.30 + 50*0.25 + 40*0.25 + 20*0.15 + 30*0.05
= 9 + 12.5 + 10 + 3 + 1.5
= 36.0
```

가중평균 산식 자체는 정확. 문제는 **LLM이 각 카테고리에 부당하게 후한 점수**:
- accuracy 50 — 실제 답변엔 기술 내용 없음, "셀레니움" 같은 키워드만 반복
- practicality 40 — 반복 문자 나열에 "실무 사례"가 있을 수 없음

## 근본 원인

현재 EVALUATOR_PROMPT는 "각 역량 0~100 독립 채점"만 지시. **답변이 반복/무의미일 때 어떻게 처리해야 할지 명시 없음**.

LLM(특히 gpt-4o-mini)은 text 안에 기술 키워드가 섞여 있으면 "일부나마 답을 했다"고 판단해 30~50점을 매기는 경향. 실질적 내용 부재를 알아채지 못함.

## 수정

### A. EVALUATOR_PROMPT에 "무의미/반복 답변" 규칙 추가
명시적으로:
- 답변 글자 수의 상당 부분이 단일/짧은 phrase 반복이면 → clarity 20 이하, 나머지도 30 이하
- 답변이 질문과 무관하거나 키워드만 나열이면 → 모든 카테고리 20 이하
- 답변이 너무 짧거나 거의 내용 없음이면 → 모든 카테고리 10 이하
- 건너뜀/"답변 못 함" 수준이면 → 모든 카테고리 0

### B. 후처리 안전장치
`_normalize_evaluation`에서 답변이 비어 있거나 정규화 후 너무 짧으면 scores 전체를 0~15 사이로 clamp. (프롬프트가 무시하는 케이스 보호)
- 구현: evaluator_agent.evaluate_answer에 answer 원문 전달 (또는 정규화 텍스트 전달) → 정규화 후 길이 체크 후 cap

### C. 버그 #2 가드와 상호보완
버그 #2 수정(짧은 답변 제출 차단)이 적용되면 이런 저품질 답변 자체가 evaluator까지 도달하지 않음. 그래도 프롬프트/후처리 보강은 안전망으로 유지.

## 영향 범위
- `backend/app/prompts/agent.py` EVALUATOR_PROMPT
- `backend/app/agent/evaluator_agent.py` `_normalize_evaluation` (가능 시 답변 텍스트 기반 cap)
- 회귀 우려: 낮은 점수가 더 자주 나와 사용자 체감 험해짐 → 정상. "개떡 답변 = 낮은 점수" 가 올바른 동작.

## 검증
- 동일 답변(위의 209자 반복 텍스트)을 evaluator에 넣어 점수 재산출. 모든 카테고리 20 이하 나오는지.
- 정상 답변(구체 사례 포함 100~300자)에는 여전히 60+ 나오는지 샘플 체크.
